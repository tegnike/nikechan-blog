## 01.png「Neuro APIの基本構造」

```
【図解タイプ】双方向通信のアーキテクチャ図
【用途】技術ブログ記事の解説図
【アスペクト比】16:9（横長）

【タイトル】"Neuro APIの基本構造"

【全体レイアウト】
左に「ゲーム / ツール連携」、右に「Neuro-sama側」を置く。
中央にWebSocketの双方向矢印を配置する。
背景はオフホワイト（#FAFAFA）で、本文中の図として読みやすくする。

【左側ブロック】
大きな角丸カード。
見出し: "ゲーム / ツール連携"
中身:
- 状態を取得
- 行動候補を登録
- 結果を実行・返却
小さなアイコン: ゲーム画面、コードエディタ、歯車
アクセント色: ブルー（#3B82F6）

【右側ブロック】
大きな角丸カード。
見出し: "Neuro-sama側"
中身:
- 状況を読む
- 行動を選ぶ
- 会話・記憶と統合
小さなアイコン: AIチップ、吹き出し、記憶ノード
アクセント色: ライムグリーン（#84CC16）

【中央の通信】
上下2本の矢印。
上の矢印（左→右）:
"context / actions/register / actions/force"
下の矢印（右→左）:
"action"
さらに左側から右側へ戻る細い補助矢印:
"action/result"
中央に小さく "WebSocket + JSON"

【下部注釈】
中央下に短く:
"画面をそのまま見せるのではなく、AIが扱える形に整えて渡す"

【スタイル】
- モダンな技術ブログ向けのフラット図解
- 日本語は読みやすいサンセリフ
- 角丸カード、薄い影、余白多め
- 図の主役は「状態・行動・結果」の流れ

【制約】
- 実在キャラクターやゲーム素材は使わない
- 1つのラベルは短く、2行以内
- 装飾を増やしすぎない
- ブログ本文幅で表示しても文字が読めるサイズ
```

## 02.png「協力型ゲーム連携の分担」

```
【図解タイプ】協力型ゲーム連携の分担図
【用途】Outer Wildsのようなリアルタイム寄りゲームで、Neuro-samaが低レベル操作を直接行わないことを説明する図
【アスペクト比】16:9（1280×720）

【タイトル】"協力型ゲーム連携の分担"

【全体レイアウト】
左に Neuro-sama、中央上に Neuroscope、中央下に Neuropilot、右に Outer Wilds、右下に Vedal を配置する。

【流れ】
- Outer Wilds → Neuroscope → Neuro-sama: 船の状態 / 状況説明
- Neuro-sama → Neuropilot → Outer Wilds: 高レベル指示 / 操作を実行
- Vedal → Outer Wilds: 人間も操作

【下部注釈】
"AIは方針を出し、専用コンポーネントがゲーム操作を橋渡しする"

【スタイル】
01.png と同じ、白背景・角丸カード・青/ライムグリーンのアクセント・薄い影のフラット図解。

【制作メモ】
画像生成ツールで作成し、ブログ用に 1280×720 にリサイズした。
矢印が大きくなりすぎないよう、small thin arrows を明示して生成した。
```
