# 画像生成プロンプト集

## thumbnail.png「サムネイル（アイキャッチ）」

```
【サイズ】1200×630px（OGP標準）

【コンセプト】
「ゲーム画面がAIに解読される瞬間」を視覚化する。
ゲームUIの断片（HPバー・カード・ターン表示）の上に、緑やシアンのコード文字列が重なって「解読」されていくイメージ。
ハッカー映画的な暗いトーン × ゲームUIのポップな断片、の混合感を狙う。

【背景】
ほぼ黒（#0A0A0F）。左上から中央にかけて、深い紫（#1E1040）の薄いグローが入る。

【背景テクスチャ（画面全体に薄く敷く）】
- 架空のゲームUIの断片を画面全体に薄く散らす。あくまで「テクスチャ」として機能させる。前に出すぎない。
  - 例：HPバーのシルエット、手札カードの輪郭、「TURN 12」「ENERGY 3」などの文字（不透明度20〜30%）
  - 明るいグリーン（#4ADE80）やシアン（#22D3EE）で「{ "hp": 68, "hand": [...] }」「API」「DOM」「MOD」などの断片が斜めにランダム配置（不透明度15〜25%）
  - 読める必要はない。雰囲気として「ゲームとコードが混在している感」が出ればOK

【前面：タイトルテキスト（最優先・大きく）】
- 画面の左半分〜中央寄りに、縦積みで大きく配置する
  1行目「AIにゲームを遊ばせるなら」（白、太字、やや細めのサンセリフ）
  2行目「状態をどう取る？」（明るいシアン #22D3EE、太字、1行目より1.4倍大きく）
- 2行目は特に主役。背景テクスチャより確実に前に来るよう、テキストに薄い黒ドロップシャドウを入れる
- タイトル下に小さくグレーで「Pokémon Showdown / Slay the Spire / Minecraft…」と並べる（記事内のゲーム名サンプル）

【キャラクター配置】
- 右下〜右端に女子VTuberキャラクター（ニケちゃん）を配置する。
- ポーズ：右向きで、背景のゲーム画面・コード群を眺めながら少し驚いている表情（「なにこれ？」的な）。
- サイズ：画像高さの75〜85%程度。右端から少しはみ出るくらいのサイズ感でもOK。
- キャラにも薄くシアンや緑のコード文字のグローが当たっているようにして背景と馴染ませる。

【スタイル】
- ダークモード基調。明るい部分は「発光」ではなくコードやUIの輝きだけにする
- フォントは角張ったサンセリフ（Bebas Neue的な縦長 or モノスペース混合）
- ゲームUIの断片は「実在ゲームを特定されないレベル」の抽象度にとどめる

【制約】
- テキストは端から120px以上内側に配置
- ブログ一覧で縮小されても「コードとゲームが混ざった画像」と判読できるようにする
- 実在ゲームのロゴや公式キャラクターは描写しない
- 日本語テキストは上記の文言のみ
```

## 01.png「AIゲームプレイに必要な3要素」

```
【図解タイプ】3ステップの処理フロー図
【用途】技術ブログ記事の解説図
【アスペクト比】16:9（横長）

【タイトル】"AIゲームプレイに必要な3要素"

【全体レイアウト】
左から右への3ステップフロー。
背景はオフホワイト（#FAFAFA）。
3つの大きな角丸カードを横並びにし、矢印で接続する。

【カード1】
見出し: "1. 状態を取得する"
補足: "画面 / DOM / JSON / API / RAM"
アイコン: 虫眼鏡 + ゲーム画面
色: ブルー（#3B82F6）

【カード2】
見出し: "2. 行動を決める"
補足: "LLM / ルール / 探索 / 記憶"
アイコン: AIの脳またはチップ
色: シアン（#06B6D4）

【カード3】
見出し: "3. ゲームへ入力する"
補足: "コマンド / API / Bot / キー入力"
アイコン: コントローラー + 送信矢印
色: グリーン（#10B981）

【下部注釈】
中央下に小さく「ログ取得だけでは終わらない」と配置。

【スタイル】
- モダンでフラットなベクター図解
- 日本語の読みやすいサンセリフ
- 見出し太字、補足は小さめ
- 矢印は細めでシンプル

【制約】
- ブログで縮小表示されても読めるサイズ
- 1カード内のテキストは2行まで
- 余計な装飾を増やしすぎない
```

## 02.png「AIに遊ばせやすいゲームと難しいゲーム」

```
【図解タイプ】分類マトリクス図
【用途】技術ブログ記事の解説図
【アスペクト比】16:9（横長）

【タイトル】"AIに遊ばせやすいゲームと難しいゲーム"

【全体レイアウト】
2軸マトリクス。
横軸: "状態取得しやすい" → "状態取得が難しい"
縦軸: "操作・計画が軽い" → "操作・計画が重い"

【配置するゲームカード】
左下寄り（遊ばせやすい）:
- Pokémon Showdown
- チェス / 将棋 / 囲碁
- Slay the Spire
- TextWorld

左上寄り（状態は取れるが計画が重い）:
- Minecraft
- Factorio
- StarCraft II
- NetHack

右上寄り（かなり難しい）:
- FPS / アクションゲーム
- MOBA
- オープンワールドゲーム

右下寄り（小さく）:
- シンプルなブラウザゲーム

【注釈】
左下エリアに小さく「AIプレイ向き」
左上エリアに小さく「専用ライブラリはあるが設計が重い」
右上エリアに小さく「画面認識・反応速度・規約が課題」

【スタイル】
- 背景: オフホワイト（#FAFAFA）
- 軸線: ダークグレー（#374151）
- 遊ばせやすい領域: 薄いグリーン（#D1FAE5）
- 設計が重い領域: 薄いオレンジ（#FEF3C7）
- 難しい領域: 薄いレッド（#FEE2E2）
- カードは白背景、角丸、薄い影

【制約】
- 実在ゲームのロゴや公式キャラクターは使わない
- テキストは短く、縮小しても読める大きさ
- 表ではなくマトリクスとして一目で分類が分かる構図にする
```
