# 画像生成プロンプト集

## thumbnail.png「サムネイル（アイキャッチ）」

```
【サイズ】1200×630px（OGP標準）
【用途】ブログ記事「生成AIなんでも展示会 Vol.5 でAIキャラのゲーム実況＆プレイを展示します」のサムネイル

【背景】
- 白〜ライトシアンの明るいテックブログ風背景
- 実在ゲーム画面や既存IPのキャラクターは使わない
- 汎用のステータスバー、カード風UI、ログ、JSON風状態表示、コントローラーアイコン、矢印で「画面認識からテキスト状態取得へ」を表現する

【レイアウト】
- 中央にニケちゃんの立ち絵を配置。右側固定配置にはしない
- 左側に大きく「AIキャラ × ゲーム実況」
- 左側の補助テキストとして「画面認識からテキスト状態取得へ」
- 下部に小さく「生成AIなんでも展示会 Vol.5」
- 右側には「画面認識」「テキスト状態」の抽象UIを配置する
- テキストは青緑とブルーを主役に、白背景でも読める太めのゴシック体にする

【制約】
- 文字は端から100px以上内側に配置
- ポケモン、モンスターボール、Nintendo/Switch、実在ゲームUI、商用ゲームのスクリーンショット、モンスターやスライム風の敵キャラは入れない
- 情報量を詰め込みすぎない
- 暗いゲームサムネ風にせず、展示紹介らしい明るくクリーンな雰囲気
- 歴代サムネイルと被るため、キャラクターを右端に大きく置く構図は避ける
```

## 02.png「旧方式の構成図」

```
【図解タイプ】システム構成図
【用途】Switchの画面をスクリーンショットとして取得し、マルチモーダルLLMに渡して次の一手と実況セリフを作る旧方式を説明する
【アスペクト比】16:9

【全体レイアウト】
左から右への処理フロー。
「ゲーム画面」→「スクリーンショット」→「マルチモーダルLLM」→「次の一手 / 実況セリフ」→「音声合成 / 操作」の順。

【強調ポイント】
- マルチモーダルLLMの箱を中心に置く
- 「画面を見て推測」というラベルを入れる
- 課題として「約20秒」「誤認識しやすい」を小さな注意ラベルで添える

【スタイル】
- 白背景、濃いグレーの本文、青緑のアクセント
- ブログ本文で縮小しても読めるシンプルなブロック図
```

## 05.png「発想の転換: テキストで状態を取る」

```
【図解タイプ】概念図
【用途】画面認識から、ゲーム状態・ログ・行動候補をテキストで渡す発想へ変わることを説明する
【アスペクト比】横長

【全体レイアウト】
左側に「画面を見る方式」、右側に「テキストで状態を取る方式」を並べる比較図。
中央に矢印を置き、「発想の転換」とラベルする。

【左側】
- ゲーム画面
- スクリーンショット
- LLMが推測
- 注意ラベル: 遅い / 見落とし

【右側】
- 状態ログ
- JSON
- 行動候補
- LLMが判断
- ポジティブラベル: 速い / 安定 / プレイと実況に使える

【スタイル】
- 余白多め、文字大きめ
- 青緑とグリーンを中心にしたクリーンなテックブログ風
```
